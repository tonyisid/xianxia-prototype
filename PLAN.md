## Phase 1：核心战斗引擎（3周）

### Week 1：基础框架 + 卡牌系统

**Day 1-2：Godot 项目搭建**
- [ ] 创建 Godot 4 项目结构（scenes/scripts/assets）
- [ ] 全局状态管理 `game_state.gd`（单例 Autoload）
  - `meta` 存档结构（灵根/境界/修为/材料/道统/神通/日志）
  - `run` 存档结构（种子/层数/牌组/HP/灵石/丹药）
  - JSON 序列化/反序列化
  - 存档版本管理
- [ ] 场景管理器（洞府/战斗/地图/商店/事件/休整 切换）

**Day 3-5：卡牌系统**
- [ ] `card_system.gd` — 卡牌数据加载与效果解析
  - 从 `data/cards.json` 加载60张功法定义
  - `mk_card(name, type, cost, effects, {el, desc, rarity})` 工厂函数
  - 效果类型枚举：dmg/blk/burn/poison/draw/heal/nextAtk
  - 蓄力型 buff 系统：`pending_buffs` 数组（类型/层数/剩余触发次数）
  - 条件型判定：打出瞬间检查（HP/手牌数/本回合状态）
- [ ] `card_view.gd` — 卡牌 UI 组件
  - 显示名称/费用/效果描述/元素标签/稀有度边框颜色
  - 可点击/不可用(费用不足)状态
  - 升级状态标识

### Week 2：战斗引擎

**Day 1-3：战斗核心**
- [ ] `combat.gd` — 战斗状态机
  - 状态：START_TURN → PLAYER_PHASE → END_TURN → ENEMY_PHASE → CHECK_RESULT
  - `start_turn()`：灵力恢复(maxSp + 加成)、格挡清零、抽牌(5 + 加成)
  - `play_card(card)`：条件判定 → 五行相生 → 费用扣除 → 蓄力buff → 神通触发 → 效果结算
  - `end_player_turn()`：清除蓄力 → 弃手牌 → 灼烧tick → 敌人行动 → 中毒tick → 生存神通 → 死亡检查
- [ ] 伤害计算公式（§8.2）：
  ```
  base = 卡牌伤害 + 相生 + 道统 + 法器 + 灵根
  result = base × 焚天 × 连环诀 × 暴击
  floor(result)
  ```
- [ ] 状态效果系统：灼烧/burn tick、中毒/poison tick、格挡/block 管理
- [ ] 牌组管理：抽牌堆/弃牌堆/手牌，shuffle，draw(n)，discard_all

**Day 4-5：战斗 UI**
- [ ] `battle_hud.gd` — 战斗界面
  - 顶部：层数进度条
  - 敌人区：名称/立绘/HP条/意图图标/状态 badge
  - 战斗日志：最近3条操作文字
  - 玩家区：HP条/灵力/格挡/状态 badge
  - 手牌区：卡牌扇形排列，可点击出牌
  - 丹药按钮（每场1次）
  - 「结束回合」按钮
- [ ] 出牌动画（卡牌飞向敌人/飞向自己）
- [ ] 伤害数字弹出（浮动文字）

### Week 3：敌人 AI

**Day 1-3：意图系统**
- [ ] `enemy_ai.gd` — 敌人意图引擎
  - 意图类型：攻击N / 攻击+格挡 / 防御 / 蓄力 / 特殊
  - 意图池：加权随机选择，不连续2回合相同（除非只剩1种）
  - 蓄力意图：本回合不攻击，下回合必然释放蓄力攻击(×2)
- [ ] 8种敌人数据定义（`data/enemies.json`）：
  - 5种普通：野狼妖/青蛇妖/藤妖/蝠群/毒蟾（含意图池+权重+特殊能力）
  - 2种精英：嗜血野猪妖/幽冥蛛后（含狂暴/产卵机制）
  - 1个Boss：巨蝠妖（含常规/狂暴双意图池 + 30%狂暴线 + 吸血）
- [ ] 蝠群双实体机制（2个独立HP目标）
- [ ] 幽冥蛛后产卵机制（召唤8HP小蛛，最多2只）
- [ ] Boss 狂暴切换（HP降到30%时立即切换意图池）

**Day 4-5：战斗流程贯通测试**
- [ ] 端到端测试：开始战斗 → 出牌 → 敌人行动 → 胜利/失败
- [ ] 测试用例：
  - 基础出牌和伤害结算
  - 灼烧/中毒 tick
  - 格挡抵扣
  - Boss 狂暴切换
  - 蛛后产卵
  - 蝠群双目标
  - 玩家死亡流程
- [ ] 修复所有战斗 bug

## Phase 4：神通系统（2周）

### Week 7-8：8种神通

**Day 1-3：神通触发引擎**
- [ ] `shentong.gd` — 神通触发器
  - 触发优先级：生存(1) → 元素连击(2) → 连招(3)
  - 装备槽位：最多2个
  - 触发点：play_card() 中效果结算前 / end_player_turn() 中死亡检查
  - 每场1次的标记管理（used_this_fight）
- [ ] 全局追踪变量：
  - `consecutive_fire_count`（跨回合，焚天用）
  - `consecutive_same_cost_count` + `last_cost`（灵力共振用）
  - `atk_played_this_turn`（连环诀用）
  - `cards_played_this_turn` + 名字计数器（破绽洞察用）
  - `damaged_this_turn`（反击壁用，但这是卡牌不是神通...移除）
  - `took_damage_this_turn`（用于未来扩展）

**Day 4-7：8种神通实现**
- [ ] **焚天**：consecutive_fire_count >= 3 时触发，伤害×2，重置计数
- [ ] **枯荣**：打出木牌 + HP<50%，额外回复3HP
- [ ] **厚土**：打出土牌后，再获得2格挡
- [ ] **灵力共振**：连续3张同费用，抽1张
- [ ] **连环诀**：一回合≥4张攻击牌，最后一张+50%伤害
- [ ] **破绽洞察**：同名牌第2张打出时，费用-1
- [ ] **不灭金身**：HP降到30%以下，获得15格挡（每场1次）
- [ ] **涅槃**：死亡时1HP复活（每场1次）

**Day 8-10：神通 UI + 测试**
- [ ] 神通装备界面（洞府内）
  - 已领悟列表，最多装备2个
  - 展示触发条件和效果
- [ ] 战斗中神通触发反馈：
  - 触发时图标闪光 + 文字提示
  - 「不灭金身」「涅槃」特殊动画
- [ ] 境界突破时的三选一选择界面
- [ ] 测试所有8种神通的触发条件
- [ ] 测试神通组合（同时装备2个的交互）
- [ ] 测试神通优先级（同事件多神通触发顺序）
