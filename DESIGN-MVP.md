## 一、项目概述

**类型：** 修仙题材卡牌肉鸽（Roguelike Deckbuilder）
**平台：** PC / Steam
**引擎：** Godot 4
**核心卖点：** 「修仙+卡牌肉鸽+洞府养成」三合一，五行相生系统，Steam无同类产品
**核心循环：** 洞府养成 → 秘境历练(肉鸽) → 材料回归 → 养成强化 → 挑战更深层

### 核心设计原则

1. **70/30法则** — 70%战斗力来自当局牌组构筑，30%来自局外养成。养成给你「开局更好」，不能「替你打」
2. **每次出牌都是决策** — 五行相生让出牌顺序有意义
3. **AI驱动生产** — 美术、音乐、音效全部AI生成，成本趋近于零
4. **EA先行** — 先发1个秘境验证核心玩法，按玩家反馈迭代

## 三、道统系统（功法流派）

道统 = 职业选择。装备后改变初始牌组和提供被动加成，最多装备**1个**。

| 道统 | 学习条件 | 被动加成 | 牌组变化 |
|------|----------|----------|----------|
| 无名散修 | 默认 | 无 | 标准初始牌组 |
| 炎火道统 | 通关1次 | ⚔️+1 | -2火鸦，+1三昧真火+1火蛇符 |
| 木灵道统 | 通关1次 | ❤️+5 🛡️+2 | -1灵草护体，+1以毒攻毒+1回春术 |
| 厚土道统 | 通关1次 | 🛡️+3 ❤️+3 | -2火鸦，+1山崩诀+1灵盾符 |
| 金锐道统 | 通关1次 | ⚔️+2 💥+5% | -1火鸦，+1金光咒+1锋刃术 |
| 水云道统 | 通关1次 | 🔮+1 | -1灵草护体，+1水波术+1寒冰符 |

**解锁规则：** 所有道统通关1次后全部解锁。灵根不锁定道统，只影响卡牌出现概率。推荐玩家选择与灵根匹配的道统（概率偏向配合初始牌组），但不强制。

## 四、境界系统

### 4.1 境界等级

```
炼气1层(0xp) → 2层(100) → 3层(200) → 4层(350) → 5层(500)
→ 6层(700) → 7层(1000) → 8层(1400) → 9层(1900)
→ 筑基初期(2500) → 中期(3200) → 后期(4000)
→ 金丹初期(5000) → 中期(6200) → 后期(7500)
```

共15个境界等级。

### 4.2 修为获取

- **自动修炼：** 每分钟+5修为（离线也算，上限8小时）
- **通关奖励：** 通关秘境获得100-300修为（按通关层数）
- **事件奖励：** 部分事件给修为

### 4.3 突破

- 消耗修为达到阈值 + 材料（灵草/矿石/妖丹）
- 突破丹：炼气层间突破消耗，配方：🌿×2 + 🔮×1
- 筑基丹：炼气→筑基/筑基层间突破消耗，配方：🌿×3 + ⛏️×2 + 🔮×2
- **突破奖励：** maxHp+5，每3层额外 maxSp+1

### 4.4 养成/肉鸽平衡

为防止养成碾压肉鸽难度，养成加成设有上限：

| 来源 | 效果 | 上限 |
|------|------|------|
| 境界HP加成 | +5/次 | 最多+50（10次突破） |
| 境界灵力加成 | +1/3层 | 最多+3 |
| 道统被动 | 固定 | 不叠加 |
| 法器 | 固定 | 只能装备1件 |

总养成加成约占最终战斗力的30%。

## 五、功法卡牌系统

**功法即卡牌。** 所有战斗中打出的牌都是功法，每张功法带五行属性。

### 5.1 卡牌数据结构

```typescript
{
  id: number;
  name: string;
  type: 'atk' | 'def' | 'skill';
  cost: number;        // 灵力消耗
  effects: {
    dmg?: number;      // 伤害
    blk?: number;      // 格挡
    burn?: number;     // 灼烧
    poison?: number;   // 中毒
    draw?: number;     // 抽牌
    heal?: number;     // 治疗
    nextAtk?: number;  // 下次攻击加成
  };
  el: 'fire' | 'wood' | 'earth' | 'water' | 'metal' | 'none';
  desc: string;
  upgraded: boolean;
}
```

### 5.2 元素定位

| 元素 | 定位 | 关联机制 |
|------|------|----------|
| 🔥 火 | 进攻型 | 灼烧、高伤害 |
| 🌿 木 | 持续型 | 中毒、回复 |
| ⛰️ 土 | 防御型 | 格挡、攻防兼备 |
| 💧 水 | 控制型 | 抽牌、攻守平衡 |
| ⚔️ 金 | 暴击型 | 高直伤、穿透 |
| 无 | 中立 | 通用辅助 |

### 5.3 初始牌组（10张）

| 卡牌 | 类型 | 费用 | 效果 |
|------|------|------|------|
| 火鸦 ×4 | 攻击·火 | 1 | 造成6伤害 |
| 灵草护体 ×3 | 防御·木 | 1 | 获得5格挡 |
| 丹火引燃 ×2 | 功法·火 | 0 | 抽1张；手牌有火牌+1灵力 |
| 药炉感悟 ×1 | 功法·无 | 0 | 每打出1张火牌获得2格挡 |

道统会替换部分卡牌（见§三）。

### 5.4 功法卡池（60张）

在现有50张基础上新增10张有机制差异的功法：

#### 新增：蓄力型（5张）

| 功法 | 属性 | 类型 | 费用 | 效果 |
|------|------|------|------|------|
| 蓄火诀 | 火 | 功法 | 1 | 下2张攻击牌+3伤害 |
| 凝毒术 | 木 | 功法 | 1 | 下2张牌附加2中毒 |
| 厚土印 | 土 | 功法 | 1 | 下2张防御牌格挡+3 |
| 蓄潮术 | 水 | 功法 | 0 | 下2张牌费用-1 |
| 磨刃诀 | 金 | 功法 | 1 | 下2张攻击牌+4伤害 |

#### 新增：条件型（5张）

| 功法 | 属性 | 类型 | 费用 | 效果 |
|------|------|------|------|------|
| 背水一战 | 火 | 攻击 | 1 | HP<30%时伤害翻倍(12→24) |

### 5.6 稀有度分布

| 稀有度 | 数量 | 出现权重 | 获取方式 |
|--------|------|----------|----------|
| 白(white) | 26 | 60% | 普通战斗 |
| 绿(green) | 26 | 35% | 普通+精英战斗 |
| 蓝(blue) | 8 | 5% | 精英战斗 |

灵根概率偏向：对应元素卡出现权重×1.5

**获取规则：**
- 普通战斗胜利 → 从池中按权重随机3张，选1（可跳过）
- 精英战斗胜利 → 从池中按权重随机4张（蓝卡权重提高），选1 + 可移除1张

### 5.7 蓄力型 buff 规则

- 蓄力效果**可叠加**（打2张蓄火诀 = 下2张攻击+6）
- 蓄力效果只被对应类型消耗（「下张攻击+3」只被攻击牌消耗，防御牌跳过）
- 蓄力效果**当回合有效**，结束回合时未消耗的蓄力清除
- 蓄力计数器：每次打出对应类型牌，计数-1，到0时清除

### 5.8 条件型判定规则

- 所有条件判定在**卡牌打出瞬间**（click时），不在结算时
- 「本回合」定义：从上一次 `endPlayerTurn()` 到当前 `endPlayerTurn()` 之间
- 例：「背水一战 HP<30%翻倍」→ 打出瞬间检查HP，不是伤害结算后
- 例：「反击壁 本回合受过伤」→ 检查本轮是否已被敌人攻击过（上回合敌人行动是否打到了你）

### 5.9 卡牌升级（休整点）

- 伤害 ×1.5（向下取整：7→10, 8→12）
- 格挡 ×1.5（向下取整）
- 灼烧 +1
- 中毒 +2
- 抽牌 +1
- 费用 -1（最低0）

## 六、五行相生系统

**只做相生，不做相克。** 相生是正反馈，容易理解，好玩。

### 6.1 相生规则

```
火生土 → 打出火牌后，下一张土牌格挡+2
土生金 → 打出土牌后，下一张金牌伤害+2
金生水 → 打出金牌后，下一张水牌抽牌+1
水生木 → 打出水牌后，下一张木牌治疗+3
木生火 → 打出木牌后，下一张火牌灼烧+2
```

### 6.2 实现方式

使用**标记位**而非 `lastElement`，每种元素的相生效果独立持续，直到被对应元素消耗：

```gdscript
# 相生就绪状态，打出元素牌后设置，被生出的元素消耗
var sheng_ready = {}  # {"fire": true} 表示火生土就绪

# 相生映射表
const SHENG_MAP = {
  "fire":  {"target": "earth", "effect": "blk",  "value": 2},
  "earth": {"target": "metal", "effect": "dmg",  "value": 2},
  "metal": {"target": "water", "effect": "draw", "value": 1},
  "water": {"target": "wood",  "effect": "heal", "value": 3},
  "wood":  {"target": "fire",  "effect": "burn", "value": 2},
}

# 打出元素牌时：
# 1. 设置本元素的相生就绪（sheng_ready[el] = true）
# 2. 检查是否有相生指向当前元素的（遍历SHENG_MAP找target==el的）
# 3. 如果有就绪标记，消耗并应用加成
```

**规则细节：**
- 无属性牌不触发相生，也不消耗相生，不改变任何标记
- 相生标记跨牌持续（打出火牌后可以隔几张再打土牌）
- 相生标记在回合结束时不清除（但战斗结束时清除）
- 同一元素的相生标记不叠加（打出3张火牌，火生土还是+2不是+6）
- 相生加成显示为绿色文字提示（如「🔥→⛰️ 火生土 格挡+2」）

### 6.3 战斗日志示例

```
打出 [火蛇符·火] → 4伤害 + 2灼烧（火生土就绪✓）
打出 [灵盾符·土] → 8格挡 + 火生土→格挡+2 = 10格挡！（火生土消耗）
打出 [金光咒·金] → 9伤害 + 土生金→伤害+2 = 11伤害！（土生金消耗）
```

## 七、神通系统（8种）

被动战斗技能，满足条件自动触发。最多装备**2个**（MVP版降低上限）。

### 7.1 获取方式

- 境界突破时三选一获得1个

### 7.2 神通触发优先级

同一事件可能触发多个神通时，按以下顺序处理：

1. **生存系**（不灭金身、涅槃）— 最高优先，涉及死亡判定
2. **元素连击系**（焚天、枯荣、厚土）— 影响伤害/格挡数值
3. **连招系**（灵力共振、连环诀、破绽洞察）— 影响资源/抽牌

同优先级内，按装备槽位顺序（槽1→槽2）处理。

### 7.3 神通列表

#### 元素连击系（3种）

| 神通 | 触发条件 | 效果 |
|------|----------|------|
| 焚天 | 连续打出第3张火属性牌（计数器跨回合） | 该牌伤害翻倍 |
| 枯荣 | 打出木牌且HP<50% | 额外回复3HP |
| 厚土 | 打出土牌后 | 再获得2格挡 |

#### 连招系（3种）

| 神通 | 触发条件 | 效果 |
|------|----------|------|
| 灵力共振 | 连续打出3张相同费用的牌 | 抽1张牌 |
| 连环诀 | 一回合打出≥4张攻击牌 | 最后一张+50%伤害 |
| 破绽洞察 | 同名牌回合内打出第2张 | 该牌费用-1 |

#### 生存系（2种）

| 神通 | 触发条件 | 效果 |
|------|----------|------|
| 不灭金身 | HP降到30%以下 | 获得15格挡（每场1次） |
| 涅槃 | 死亡时 | 1HP复活（每场1次） |

## 八、战斗系统

### 8.1 战斗流程

```
startTurn()
  → 灵力恢复(maxSp + 道统/境界加成)、格挡清零
  → 抽牌（5张 + 道统/灵根加成）
  → 敌人选意图（从意图池加权随机，显示给玩家）
  → 神通回合初始化

[玩家出牌阶段]
  → 点击卡牌 → 条件判定(打出瞬间)
  → 五行相生检查(设标记+消耗标记)
  → 费用扣除
  → 蓄力buff检查并应用
  → 神通触发检查(生存→元素→连招优先级)
  → 效果结算(伤害/格挡/灼烧/中毒/抽牌/治疗)
  → 可用丹药（每场1次，点击丹药栏）
  → 点击「结束回合」

endPlayerTurn()
  → 清除未消耗的蓄力buff
  → 弃手牌
  → 灼烧tick（敌人受伤害+层数-1）
  → 敌人行动（执行意图）
  → 中毒tick（敌人受伤害+层数-1）
  → 生存系神通检查(不灭金身/涅槃)
  → 检查玩家死亡
```

### 8.2 伤害计算公式

```
step 1: 加法修正
base = 卡牌基础伤害
  + 五行相生(如土生金+2)
  + 道统被动(如炎火+1)
  + 法器(如玄铁剑+3)
  + 灵根加成(火灵根+1)

step 2: 乘法修正(按顺序)
result = base
  × 焚天(第3张火 → ×2)
  × 连环诀(≥4张攻击 → ×1.5)
  × 暴击(金灵根10% → ×2)

step 3: 最终取整
floor(result)
```

所有乘法修正独立判定，可叠加（焚天×暴击 = ×4）。

### 8.3 状态效果

| 状态 | 触发时机 | 效果 |
|------|----------|------|
| 灼烧 | 回合结束 | 减HP，层数-1 |
| 中毒 | 敌人行动后 | 减HP，层数-1 |
| 格挡 | 回合开始清零 | 抵挡伤害 |

### 8.4 丹药（战斗消耗品）

进入秘境前选择携带1种，每场战斗可用1次。

| 丹药 | 效果 |
|------|------|
| 回春丹 | 回复15HP |
| 凝元丹 | 下次攻击+5 |
| 护脉丹 | 获得10格挡 |

## 九、秘境系统

### 9.1 幽冥谷

**背景：** 千年前一位修士在此渡劫失败，魂魄碎裂散入谷中。所有敌人的「脸」都是模糊的。

**12节点，分支地图：**

```
        ┌──[F]──[E]──┐
[S]─[F]─┤            ├─[R]─[F]─[E]─[B]
        └──[?]──[S]──┘

F=战斗(6)  E=精英(2)  R=休整(1)  S=商店(2)  ?=事件(1)  B=Boss(1)
```

### 9.2 地图生成约束

每层2-3个节点选1个，按以下约束随机生成：

| 层数 | 节点类型约束 |
|------|-------------|
| 1 | 固定战斗 |
| 2 | 战斗（至少1个），0个精英 |
| 3 | 战斗+精英，或战斗+商店 |
| 4 | 战斗（至少1个），最多1个精英 |
| 5 | 固定商店 |
| 6 | 战斗+事件，或战斗+战斗 |
| 7 | 战斗（至少1个），最多1个精英 |
| 8 | 固定休整 |
| 9 | 战斗+精英 |
| 10 | 战斗（至少1个） |
| 11 | 固定事件 |
| 12 | 固定Boss |

**生成规则：**
- 地图种子用 `randi()` 生成，保存在局内存档中
- 每层可选节点数：2或3（随机）
- 节点类型按权重随机：战斗(50%) > 精英(15%) > 商店(10%) > 事件(15%) > 休整(10%)
- 固定节点不受权重影响
- 相邻层不能出现相同类型的所有选项（保证有选择差异）

### 9.3 节点奖励

| 节点 | 奖励 |
|------|------|
| 普通战斗 | 3选1功法 + 🌿×2 + 灵石×15 |
| 精英战斗 | 4选1功法 + 移除1张 + 🌿×3 ⛏️×2 🔮×1 + 灵石×30 |
| 休整 | 三选一：休息(回复30%HP) / 升级卡牌 / 精炼(移除卡牌) |
| 商店 | 移除牌(50灵石) / 升级牌(30灵石) / 买牌(40灵石) |
| 事件 | 随机5种（见§十） |
| Boss | 通关结算 + 法器3选1 |

### 9.4 灵石经济

**灵石：** 单局货币，不通关则消失。

| 来源 | 数量 |
|------|------|
| 普通战斗 | +15 |
| 精英战斗 | +30 |
| 事件(平均) | +20 |

一局总计约可获得 140-180 灵石（6普通+2精英+1事件 = 90+60+20 = 170），够买3-4样东西。

### 9.5 法器（通关奖励）

Boss通关后三选一：

| 法器 | 效果 |
|------|------|
| 玄铁剑 | 攻击牌+3伤害 |

## 十、事件系统（5种）

地图中的「?」节点，随机触发1种。

| 事件 | 描述 | 选项 |
|------|------|------|
| 仙人指路 | 遇到一位老道士，指引你三条路 | A)获得1张蓝(稀有)功法 B)获得🌿×5 ⛏️×3 C)获得100修为 |
| 赌局 | 与赌仙对赌 | A)押50灵石(50%概率赢150) B)不赌，得30灵石 |
| 宝箱 | 发现古修洞府 | A)直接开(70%获得蓝功法，30%陷阱-10HP) B)小心打开(100%获得绿功法) |
| 心魔 | 心魔试炼 | 镜像战斗（见下方规则），赢了获2张稀有功法 |
| 祭坛 | 远古祭坛 | A)献祭1张牌，获得1张蓝(稀有)功法 B)献祭10HP，获得+5maxHp(受养成上限约束，见§4.4) |

**心魔事件详细规则：**
- 心魔 = 特殊敌人，HP = 玩家当前maxHP
- 心魔使用玩家**当前牌组的复制品**（不是初始牌组），随机选牌的效果作为敌人行动
- 意图：随机从玩家牌组中抽1张，展示其效果（如「火鸦 6伤害」）
- 心魔没有灵力限制，每回合执行1张随机牌的效果
- 心魔战斗中**不能使用丹药**
- 心魔没有格挡/灼烧/中毒（简化实现）
- 胜利奖励：2张蓝(稀有)功法选1张

**祭坛事件约束：**
- 献祭10HP获得的+5maxHp受§4.4养成上限约束（总HP加成不超过+50）
- 如果已达上限，选项B变为「献祭10HP，获得50灵石」

## 十一、敌人系统

### 11.1 敌人意图系统

每个敌人有一组意图（intent），每回合开始时从意图池中加权随机选择一个，显示给玩家。

**意图类型：**
| 意图 | 图标 | 说明 |
|------|------|------|
| 攻击N | ⚔️ | 造成N伤害 |
| 攻击+格挡 | ⚔️🛡️ | 造成伤害并获得格挡 |
| 防御 | 🛡️ | 获得格挡 |
| 蓄力 | 💫 | 下一次攻击伤害×2 |
| 特殊 | ⚡ | 敌人专属技能（召唤、治疗等） |

**意图规则：**
- 意图在玩家回合开始时确定并显示
- 蓄力意图：该回合不攻击，下一回合必然释放蓄力攻击
- Boss的意图池更复杂，包含阶段切换（狂暴线以下/以上用不同池）
- 敌人不会连续2回合使用相同意图（除非只剩1种）

### 11.2 普通敌人（5种）

### 存档结构

两层存档：永久进度(meta) + 当局肉鸽状态(run)。

```json
{
  "version": 1,
  "meta": {
    "linggen": "fire",
    "realm": 3,
    "xiuwei": 150,
    "offlineXiuweiAccum": 0,
    "lastOnline": 1746000000000,

    "kuangshi": 5,
    "yaodan": 3,
    "poJingDan": 2,
    "zhuJiDan": 1,

    "unlockedDaotong": ["basic", "yanhuo"],
    "unlockedShentong": ["fentian", "niepan"],
    "equippedShentong": ["fentian"],
    "collectedLogs": [1, 3, 5],
    "hardModeUnlocked": false,
    "equippedArtifact": {"id": "sword", "name": "玄铁剑"},
    "gongfaCollection": ["火蛇符", "金光咒"],
    "runs": 15,
    "wins": 3
  },
  "run": {
    "active": true,
    "seed": 12345,
    "currentLayer": 5,
    "mapNodes": [...],
    "deck": [...],
    "hp": 35,
    "maxHp": 45,
    "lingshi": 80,
    "pill": "回春丹",
    "usedPillThisFight": false,
    "layerHP": [45, 45, 40, 38, 35]
  }
}
```

**存档规则：**
- `meta` 每次洞府操作后自动保存
- `run` 每场战斗后自动保存，支持断点续玩
- `run.active = false` 表示不在秘境中
- `run = null` 时 `meta` 完整保存，无数据丢失
- 版本迁移：`version` 字段用于未来存档格式变更
